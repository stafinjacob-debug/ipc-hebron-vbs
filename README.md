# IPC Hebron VBS

Vacation Bible School operations app: seasons, registrations, children, volunteers, and role-based staff access. **Product requirements:** [`docs/vbs-requirements.md`](docs/vbs-requirements.md). **Secrets map:** [`docs/CREDENTIALS-AND-CONFIG.md`](docs/CREDENTIALS-AND-CONFIG.md). **Implementation rules:** [`.cursor/rules/vbs-app-rules.md`](.cursor/rules/vbs-app-rules.md).

## Stack

- **App:** Next.js (App Router) in [`web/`](web/)
- **DB:** PostgreSQL + Prisma ORM (Prisma 7 with `@prisma/adapter-pg`)
- **Auth:** Auth.js (NextAuth v5) — credentials provider, JWT sessions
- **Production:** Azure App Service (Linux) + Azure Database for PostgreSQL Flexible Server (recommended)

## Optional: Azure PostgreSQL for local development

If you use an **Azure Database for PostgreSQL** instance instead of Docker, put its connection string in `web/.env.local` as `DATABASE_URL` (with `sslmode=require`). A helper file `web/.env.azure.local` may be generated by provisioning scripts; it is gitignored—merge its `DATABASE_URL` into `.env.local` and add `AUTH_SECRET` / `AUTH_URL` there as usual.

**Firewall:** Your home IP must be allowed on the flexible server (Azure adds a rule when you use `--public-access <ip>`). If your IP changes, add a new rule with `az postgres flexible-server firewall-rule create`.

## Local setup (macOS)

1. **Prerequisites:** Node.js 22+, Docker Desktop (optional, for Postgres).

2. **Database**

   ```bash
   cd web
   docker compose up -d
   ```

   Or use any PostgreSQL instance and note the connection string.

3. **Environment**

   ```bash
   cp .env.example .env.local
   ```

   Set `DATABASE_URL`, `AUTH_SECRET` (`openssl rand -base64 32`), and `AUTH_URL=http://localhost:3000`. For seeding, set `SEED_ADMIN_PASSWORD` (and optionally `SEED_ADMIN_EMAIL`).

4. **Install, migrate, seed, run**

   ```bash
   npm install
   npm run db:migrate
   npm run db:seed
   npm run dev
   ```

   Open [http://localhost:3000](http://localhost:3000). Sign in with the seeded admin email and `SEED_ADMIN_PASSWORD`. A sample volunteer account uses `volunteer@example.local` with the same password.

| `npm run build:azure` | Build + package standalone output for App Service |

## GitHub → Azure CI/CD

The workflow [`.github/workflows/deploy-azure-webapp.yml`](.github/workflows/deploy-azure-webapp.yml) builds the standalone bundle and deploys `web/.next/standalone` using a publish profile.

1. Create **Azure Database for PostgreSQL** and **App Service (Linux, Node 22)**.  
2. Configure App Service **Startup Command:** `node server.js` (deployed package root must contain `server.js`).  
3. Set application settings: `DATABASE_URL`, `AUTH_SECRET`, `AUTH_URL`, `NODE_ENV=production`.  
4. Add GitHub Actions secrets per [`docs/CREDENTIALS-AND-CONFIG.md`](docs/CREDENTIALS-AND-CONFIG.md).  
5. Push to `main` (or run the workflow manually).

**Rollback:** Re-run a previous successful workflow from the Actions tab, or introduce deployment slots later for swap.
